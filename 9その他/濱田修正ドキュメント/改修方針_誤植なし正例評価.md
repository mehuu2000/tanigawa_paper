# 誤植なし正例評価に向けた改修方針

## 最終目標

現在の `卒業研究_谷川明英` を修正し、誤植なし正例データを用いて評価を行い、その結果データを確認する。

## 基本方針

* 可能な限り現在の谷川さん実装をそのまま使う。
* まずは誤植ありの既存評価を再現し、その後に誤植なし正例へ切り替える。
* いきなりロジックを変えず、実行前提の復元と入出力データの切替を優先する。

## 事前確認で分かったこと

* 評価本体は `4プログラム/evaluation_experiment.ipynb`。
* 単発確認は `4プログラム/execute_proposed_method.ipynb`。
* 評価データ生成は `4プログラム/create_references.ipynb`。
* Solr 登録は `4プログラム/jalc-to-solr.ipynb`。
* 現在のノートブックは `eval_experiment_data/...` や `pre_experiment_data/...` を読む前提だが、実ファイルは `5データ/...` 配下にある。
* ノートブックは `from tools import wakati, generate_query` を前提にしているが、現リポジトリには `tools/` ディレクトリが見当たらない。
* `wakati.py` は `8プロジェクト関連/8-3共有プログラム/` に存在する。
* 評価実験用の誤植あり正例は既にあるが、同じ母集団の誤植なし正例ファイルは現状では確認できていない。

## 実施ステップ

### 0. 実行前提の復元

目的:
ノートブックを現在のリポジトリ上で動かせる状態にする。

確認・対応対象:

* `tools` import の前提
* `eval_experiment_data` / `pre_experiment_data` の相対パス
* 実行ディレクトリ依存のファイル参照

注意:
この段階では評価ロジック自体は変えない。

### 1. 環境構築とデータベース再構成

目的:
谷川さん実装に合わせて MongoDB / Solr 側を揃える。

作業:

* MongoDB に JaLC 2024 年版データを登録
* Apache Solr の core `jalc` を谷川さん方式で再作成
* `jalc-to-solr.ipynb` を用いて JaLC 論文メタデータを再登録

理由:
現在の Solr 登録内容は自分の解釈に基づく再実装であり、谷川さん方式の再現確認には向かないため。

### 2. 誤植ありの既存評価を再現

目的:
まず論文に近い評価結果が出るかを確認する。

作業:

* 既存の評価実験用データを使って `evaluation_experiment.ipynb` を実行
* 誤植あり正例:
  `5データ/5-2評価実験用データ/5-2-1参考文献文字列（実在）/positive_reference_*_eval.txt`
* 負例:
  `5データ/5-2評価実験用データ/5-2-2参考文献文字列（架空）/negative_reference_*_eval.txt`
* DOI 対応表:
  `5データ/5-2評価実験用データ/5-2-1参考文献文字列（実在）/doi_list.txt`

確認事項:

* 論文記載値にどの程度近いか
* どこでズレるか
* ズレの原因がデータ、環境、Solr、またはノートブック修正にあるか

### 3. 誤植なし正例データの準備

目的:
正例の誤植有無だけを変えて再評価できる状態にする。

現時点の見立て:

* 既存の `positive_reference_*_eval.txt` は誤植あり。
* `positive_reference_real.txt` は存在するが件数が異なり、評価実験用の 863 件セットとは別物。
* したがって、同一母集団の誤植なし正例は新たに作る可能性が高い。

作業候補:

* `create_references.ipynb` を確認し、評価実験用 863 件と同じ母集団から誤植なし版の正例を生成する
* 可能なら「誤植あり生成」と「誤植なし生成」を分岐できるようにする

重要:
この段階では負例は原則そのまま据え置く。

### 4. 誤植なし正例で再評価

目的:
誤植なし正例にしたときの評価結果を確認する。

作業:

* `evaluation_experiment.ipynb` の正例入力だけを誤植なし版へ切替
* 負例は従来データを利用
* RC / CC / MC / BM25 系の結果を確認

確認したい点:

* 正例側の正常検出数がどう変わるか
* 検出漏れや誤検出がどう変化するか
* 誤植の有無がどのスコアリング方式に強く効くか

## 推奨する進め方

1. まず Step 0 を行う。
2. 次に Step 1 で Solr を谷川さん方式に揃える。
3. Step 2 で baseline を確認する。
4. baseline が確認できたら Step 3 で誤植なし正例を作る。
5. 最後に Step 4 で再評価する。

## 注意点

* 最初から「誤植なし正例評価」だけを先にやると、結果差が実装差なのかデータ差なのか判別しづらい。
* 先に誤植あり評価を再現しておくと、その後の比較がしやすい。
* 現状では「評価対象ファイルの切替だけ」で済む保証はない。
  特に、誤植なし正例の同一母集団版が未整備の可能性があるため。

## 当面の最初の作業

最初に着手するべきなのは、`evaluation_experiment.ipynb` と `create_references.ipynb` の実行前提を揃えるための確認である。
具体的には、`tools` import と評価データの相対パスの扱いを先に整理する。
